# Overview

## Requirements

### Docker

To run a Lagoobernetes Project, your system must meet the requirements to run Docker. We suggest installing the latest version of Docker for your workstation. You can download Docker [here](https://www.docker.com/get-docker). We also suggest allowing Docker at least **4 CPUs** and **4 GB RAM**.

### Local Development Environments

TL;DR: install and start `pygmy`:

```text
gem install pygmy
pygmy up
```

[Pygmy](https://pygmy.readthedocs.io/en/master/) is an amazee.io flavored local development system.

Learn more about Lagoobernetes, pygmy, and [Local Development Environments](local_development_environments.md)

## Overview of Lagoobernetes Configuration Files

### `.lagoobernetes.yml`

This is the main file that will be used by Lagoobernetes to understand what should be deployed, as well as many other things. See [Documentation for .lagoobernetes.yml](lagoobernetes_yml.md)

### `docker-compose.yml`

This file is used by `Docker Compose` to start your local development environment. Lagoobernetes also uses it to understand which of the services should be deployed, which type, and how to build them. This happens via `labels`. See [Documentation for docker-compose.yml](docker-compose_yml.md)

### Dockerfiles

Some Docker images and containers need additional customizations from the provided images. This usually has two reasons:

1. **Application code**: Containers like Nginx, PHP, Node.js, etc., need the actual programming code within their images. This is done during a Docker build step, which is configured in a Dockerfile. Lagoobernetes has full support for Docker, and therefore also allows you full control over the resulting images via Dockerfile customizations.
2. **Customization of Images**: Lagoobernetes also allows you to customize the base images according to your needs. This can be to inject an additional environment variable, change a service configuration, or even install additional tools. We advise caution with installing additional tools to the Docker images, as you will need to maintain any adaptions in the future!

All images are pushed to [https://hub.docker.com/u/amazeeio](https://hub.docker.com/u/amazeeio). We suggest always using the latest tag \(like `amazeeio/nginx:latest`\) or unsuffixed images \(like `amazeeio/node:10`\), as they are kept up to date in terms of features and security. If you choose to use a specific Lagoobernetes version of an image like `amazeeio/nginx:v0.21.0` or `amazeeio/node:10-v0.21.0` it is your own responsibility to upgrade the version of the images as soon as a new Lagoobernetes version is released!
